# Contributing to chagg-github-actions

Thanks for considering a contribution! Here's how to get started.

## Getting started

1. Fork the repo and clone it locally.
2. The actions are pure YAML (composite actions) — no build step required.

## Making changes

- Keep PRs focused — one logical change per PR.
- Edit the `ci/action.yml` or `release/action.yml` files directly.
- Test changes by running the action in a GitHub Actions workflow.

## Pull requests

- Open a PR against `main`.
- Assign **@bitionaire** if you'd like a review.
- Describe *what* changed and *why*.

## Reporting bugs & requesting features

Use [GitHub Issues](https://github.com/codested/chagg-github-actions/issues).

## Code of conduct

See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md). Short version: be respectful, stay on topic, keep it about the code.
